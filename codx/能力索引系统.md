## 5. 常用工作流模板

### 📝 小说创作工作流（Stage 0~5）
```
Stage 0：灵感捕获 → 一句话概念
Stage 1：大纲推演 → 三幕结构/人物弧光
Stage 2：三遍写作 → 骨架→血肉→神经
Stage 3：三遍优化 → 语言→节奏→意境
Stage 4：冷置 → 放置24h+重新通读
Stage 5：风格宪法检查 → 反陈词滥调→角色差异化→感官描写

内置检查点：
├─ 热叙事底：开篇三句话有温度
├─ 口语魂：对话自然，不书面化
├─ 反陈词滥调：负空间/异常细节/人物化
├─ 角色差异化：每个人物说话方式不同
└─ 情绪沉淀点：高潮后物象哲思停顿
```

### 🛡️ 安全扫描工作流
```
Phase 1：威胁建模 → 系统边界/资产/威胁/缓解
Phase 2：发现漏洞 → 自动化扫描+手动分析
Phase 3：验证 → 确认真实漏洞/误报
Phase 4：攻击路径分析 → Source到Sink完整链路
Phase 5：输出报告 → Markdown报告+修复建议
```

### 🚀 GitHub发布工作流
```
1. 确认范围 → git status/分支检查
2. 提交 → git add + commit
3. 推送 → git push origin branch
4. 创建PR → yeet skill（Draft PR）
5. 处理评论 → gh-address-comments
6. 合并 → PR合并完成
```

### 🔧 问题排查工作流
```
1. 复现问题 → 确认现象/环境
2. 收集信息 → 日志/错误信息/堆栈
3. 定位根因 → 二分法/排查法
4. 修复 → 最小修改原则
5. 验证 → 回归测试/确认修复
6. 总结 → 记录到知识库
```
