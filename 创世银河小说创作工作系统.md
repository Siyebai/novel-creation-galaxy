## 序章：系统哲学

### 双螺旋核心

```
工艺螺旋（v6.0精髓）          商业螺旋（V3.0 Pro精髓）
    ↓                              ↓
风格精度 · 质量门禁           市场定位 · 平台基因
工具链 · 反AI检测            情绪工程 · 数据校准
角色语音 · 冷热切换           爽点放大 · 防崩盘
    ↓                              ↓
        交汇 → 创世银河引擎 ← 交汇
```

**根本信念**：用最理性的工程框架承载最疯狂的个人灵感；用最符合神经科学的多巴胺泵输送独一无二的灵魂底色。

### 如何使用本系统

**开新书（长篇）**：从第一卷到第六卷通读，每卷执行对应步骤。
**短篇投稿（知乎/豆瓣）**：直接跳到附录 E「短篇/中篇创作轨道」，使用精简流程。
**连载中**：直接跳到第三卷（风格校准）和第六卷（数据诊断）。
**卡文时**：第二卷（情绪曲线检查）+ 第五卷（写前SOP）。
**完本后**：第六卷（数据复牌）+ 第三卷（风格复盘）。

## 第一卷：战略定位层

> 拒绝盲目开书。先搞清楚"你是谁"和"市场要什么"。

### 1.1 认知原点深潜——找到你的母题

列出你人生中最刻骨铭心的 3 次情绪体验（极致的屈辱、狂喜、恐惧）。**这是你永远写不腻的情绪燃料**。

```
母题诊断公式：
  我的三次极致体验：_________ + _________ + _________
  → 共同的情绪内核：_________
  → 这个内核 → 赛道映射
```

| 母题情绪 | 适配赛道 | 典型作品模式 |
|---------|---------|------------|
| 被剥夺后的复仇 | 逆袭打脸 / 强者归来 / 废柴逆天 | 退婚流、重生复仇 |
| 无力感与守护 | 克苏鲁/规则怪谈 / 治愈救赎 | 诡异流、温情日常 |
| 被误解的孤独 | 独行侠 / 反英雄 / 幕后黑手 | 影帝流、马甲流 |
| 对秩序的反叛 | 反套路 / 颠覆世界观 / 第四天灾 | 玩家流、造反流 |
| 对未知的渴望 | 探索 / 解密 / 诸天无限 | 无限流、考古流 |
| 对归属的追寻 | 家族 / 宗门 / 团队成长 | 家族流、学院流 |

**操作**：将母题情绪写入 `idea_seed.md` 的 `emotional_core` 字段。后续所有创作决策——从人物设计到爽点选择——都从这个母题出发。

### 1.2 欲望满足法 × 趋势发想法

**不要拍脑袋想创意**。用以下公式：

```
欲望满足法（读者在现实中缺什么）
  + 趋势发想法（当下什么话题在爆发）
  = 高潜力题材

示例：
  "AI 焦虑"（趋势）+ "被剥夺感"（母题）= 《我的AI伴侣被财阀格式化了》
  "内卷无力"（欲望）+ "守护母题"（母题）= 《我在诡异世界开庇护所》
  "职场PUA"（欲望）+ "反叛母题"（母题）= 《废柴律师用规则系异能反杀》
```

### 1.3 平台基因适配

不同平台是不同的生态系统。**开书前必须选定主发平台**，并按对应基因设计参数。

#### 番茄/七猫（免费平台）

| 参数 | 设定 | 原因 |
|------|------|------|
| 开篇节奏 | 前300字完成情境+情绪+悬念三重爆破 | 流量筛选，完读率至上 |
| 冲突密度 | 每300字设微型冲突或信息增量 | 防止划走 |
| 标题策略 | 数值锁定法（《九千万亿人民币》）或矛盾对冲式（《新规出台，小说到此为止了？》） | 点击率驱动 |
| 语言门槛 | 初中生无障碍阅读 | 下沉市场 |
| 章节长度 | 2000-3000字/章 | 广告插入节奏 |
| 情绪曲线 | 快频震荡，虐不过半章 | 碎片化阅读 |
| 金手指 | 开局即激活，不许藏 | 读者耐心≤3章 |

#### 起点（付费/IP平台）

| 参数 | 设定 | 原因 |
|------|------|------|

| 钩子密度 | 每章至少1个悬念/秘密/真相碎片 | 长期追读 |

| 节奏 | 允许慢热，但不能"水" | 付费意愿与内容密度正相关 |
| 风格 | 允许文青，允许实验 | 差异化竞争 |
| 章节长度 | 3000-5000字/章 | 付费性价比感知 |
| 成长弧 | 必须有清晰的能力阶梯和代价体系 | 长期陪伴感 |

#### 晋江（女性向/情感平台）

| 参数 | 设定 | 原因 |
|------|------|------|
| 优先级 | **人设 > 剧情** | 情感驱动消费 |
| 人设公式 | 鲜明标签 + 反差萌 + 成长空间 | 嗑CP的前提是喜欢人 |

| 禁忌 | 男频绿帽 / 女频种马 / 主角三观不正 | 一票否决 |
| 语言 | 可精致可口语，但必须"有感觉" | 审美驱动 |

**操作**：选定平台后，将对应参数写入 `idea_seed.md` 的 `platform_config` 字段。

### 1.4 竞品扫描——赛博扫榜

在选定平台，搜索与你母题相近的 TOP 50 作品：

1. 记录它们的**前3章开篇手法**（情境/冲突/金手指出现时机）
2. 记录它们的**情绪曲线形态**（快频震荡 vs 波浪渐进）

## 第二卷：故事架构层

> 大纲不是事件流水账。先画情绪折线，再填剧情；先装发动机，再造车身。

### 2.1 情绪抑扬波——卷级宏观设计

**核心法则**：相邻高点和低点必须截然相反。虐完必甜，大爽后必小憋屈。

```
情绪折线图示例（一卷10章）：
  +3 ┤         ╱╲        ╱╲
  +2 ┤   ╱╲  ╱    ╲    ╱    ╲
  +1 ┤ ╱    ╲╱      ╲╱        ╲
   0 ┤╱                          ╲──
  -1 ┤                              ╲
  -2 ┤
      Ch1 Ch2 Ch3 Ch4 Ch5 Ch6 Ch7 Ch8 Ch9 Ch10
      开场 小爽 压抑 爆发 缓冲 悬念 绝望 翻盘 余韵 钩子
```

**设计规则**：
- 正向极值(+3)：全书级别的大爽点，一卷最多1-2处
- 正向(+1/+2)：阶段性胜利、关系进展、能力突破
- 零位(0)：信息传递、过渡、日常
- 负向(-1/-2)：挫折、失去、背叛。**最痛处用最冷的笔法写**（见第三卷冷热切换）
- 禁止：+3→+3 连续高潮（情感透支）、-2→+3 跳跃（冰火两重天）、0→+3 无铺垫跳跃
- 黄金比例：一卷内高低点比 ≈ 6:4（60%上行或缓冲 + 40%下行或紧张）
- 卷末必须：钩子（悬念/新目标/新威胁）→ 驱动翻页

**操作**：在 `beat_sheet.md` 中先画出全卷情绪折线，标注每个高低点的情绪类型和强度，再往里面填入具体事件。

### 2.2 四轮驱动引擎——长篇不崩的底层动力

确保 50 万字后不塌，必须构建四个循环驱动的引擎：

```
        ┌──────────────┐
        │  实力提升引擎  │ ← 60% 驱动力
        │  (境界/技能)   │    每次突破立即设置更强对手
        └──────┬───────┘
               │ 新能力 → 新挑战
        ┌──────┴───────┐
        │  世界扩展引擎  │ ← 20% 驱动力
        │  (塔状地图)    │    城市→大陆→星域→宇宙
        └──────┬───────┘    每换地图，对手/资源/秘密升级
               │ 新领域 → 新规则
        ┌──────┴───────┐
        │  矛盾冲突引擎  │ ← 驱动张力的燃料
        │  (多层嵌套)    │    外部(敌人/势力) + 内部(心魔/情感)
        └──────┬───────┘    + 成长冲突(获得力量必须付出代价)
               │ 冲突 → 行动
        ┌──────┴───────┐
        │  爽点释放引擎  │ ← 回报读者的时刻
        │  (压抑→爆发)   │    严格遵循：压抑→积累→爆发→收尾
        └──────────────┘    压抑越深，爆发越爽
```

**实力提升引擎的节奏模板**（适用于升级流）：

| 阶段 | 字数 | 做什么 | 配套 |
|------|------|--------|------|
| 炼气期 | 5-10万 | 快速打基础，展示世界规则 | 金手指激活，首个敌人建立 |
| 筑基期 | 10-15万 | 初步崛起，第一个小高潮 | 收第一个同伴，建立初步势力 |
| 金丹期 | 15-25万 | 区域称霸，世界观大幅扩展 | 换第一张地图，对手层级升级 |
| 元婴期 | 25-40万 | 跨域争锋，卷入更大棋局 | 揭示世界观真相碎片 |
| 化神+ | 40万+ | 世界之巅，终极对抗 | 所有伏笔收束 |

**非升级流替代方案**（规则怪谈/悬疑/日常向）：

| 替代引擎 | 替代实力提升 | 节奏模式 |
|---------|------------|---------|
| 规则破解引擎 | 每关新规则 + 推理升级 | 理解规则→利用规则→打破规则→新关 |
| 关系深化引擎 | 每事件推进关系 + 角色层次 | 初识→信任→裂痕→和解→更深的信任 |
| 真相揭示引擎 | 每卷揭示一层真相 + 代价 | 得到线索→误解→再得线索→拼图完成 |

### 2.3 三线钩悬念矩阵

让读者处于"解谜漩涡"中，停不下来：

```
第一线·现实钩（开篇即埋）
  → 发生在当下的异常/冲突/秘密
  → 例如：主角收到的匿名信 / 每天重复的同一天 / 监控中的时间错位
  → 策略：每 3 章给一点线索，每 10 章给一次重大突破

第二线·历史钩（第5-10章埋入）
  → 世界/角色的过去秘密
  → 例如：上古战争真相 / 父母失踪的原因 / 能力体系的来源
  → 策略：通过遗迹/老人/文献碎片渐进揭示，永远只给70%真相

第三线·未来钩（第15-20章埋入）
  → 关于终局走向的暗示
  → 例如：预言之子的双重身份 / 最终BOSS早早登场但伪装
  → 策略：用反差/误导/不可靠叙述保护真相，在60-70%处首次揭露
```

**三线进度管理**（写入 `plot_threads.md`）：

| 钩线 | 埋入章 | 首次线索 | 大突破 | 完全揭示 | 状态 |
|------|--------|---------|--------|---------|------|
| 现实钩 | 1 | 3 | 10 | 20 | □ |
| 历史钩 | 8 | 12 | 25 | 40 | □ |
| 未来钩 | 18 | 25 | 45 | 60 | □ |

### 2.4 难度梯度设计

每次新挑战的难度比上一次高 **20-30%**。这个区间经过验证：低于20%读者觉得没进步，高于30%读者觉得不合理。

```
难度曲线公式：
  D_n = D_{n-1} × (1.2 ~ 1.3)
  
  其中 D = 对手实力 / 破局所需资源 / 时间紧迫度 / 道德困境深度
  四个维度不必同时递增，但至少2个维度每次递增
```

**难度类型轮换**（防止重复感）：
- 第1关：力量对抗（纯粹的强弱对决）
- 第2关：规则博弈（在规则内取胜，不是蛮力）
- 第3关：信息不对称（谁掌握信息谁赢）
- 第4关：道德困境（赢了但代价巨大）
- 第5关：多重博弈（三方以上，没有纯粹敌人）

## 第三卷：风格锻造层 ★

> 这是将你从"写手"推向"不可替代"的核心。让文字拥有指纹。

风格不是"文笔好"。风格是一套**可参数化、可执行、可检测**的语言决策系统。

### 3.1 风格宪法——五条铁律

这些不讨论，不妥协：

**铁律一：禁止AI高频词**

以下词汇和它们的近亲永远不要出现在正文中：

```
心中一惊 / 眼中闪过 / 不由得 / 似乎 / 仿佛 / 微微一笑 / 沉声道
淡淡道 / 冷声 / 眸光 / 唇角 / 身形 / 眼底 / 眸色 / 指尖
涌起一股 / 难以言喻 / 前所未有 / 深深的 / 一种...的感觉
```

违反 → `novel deslop scan` 标记 CRITICAL，必须清零。

**铁律二：对话标签只用"说/问/答/喊/叫"**

偶尔用动作打断（"他笑了一下""她沉默了"）。绝对禁止：沉声道、淡淡道、冷声道、幽幽道、咬牙切齿道、一字一顿道……

原因：X道标签是作者在替角色解释情绪，而情绪应该通过对话内容和身体反应自己呈现。

**铁律三：情绪不空洞——生理唤醒原理**

```
禁止：他很生气 / 她很伤心 / 他感到恐惧
必须：至少2种身体反应 + 1种物质细节

示例：
  "她下颌收紧。指节发白。指甲在手心掐出四个月牙。"
  "他的膝盖在抖。不是怕——他自己都没注意到。"
```

**铁律四：每句话默念**

写得顺口比写得好看重要十倍。不顺口的改。这条没有例外。

**铁律五：遮名可辨**

遮住角色名，纯看对话，能分清谁在说话。分不清 → 角色语音没有区分度 → 回到第四卷重做角色语音卡。

### 3.2 五级温度模型——冷热切换系统

最痛的场景用最冷的方法写。哭不出来比放声大哭更有力。

| 级别 | 名称 | 句长(字) | 叙事距离 | 情绪暴露度 | 使用场景 |
|------|------|---------|---------|-----------|---------|
| +2 | 滚烫 | 15-28 | 极近（读心） | 100% 直接释放 | 正向高潮：胜利/重逢/狂喜/告白 |
| +1 | 温热 | 12-22 | 近 | 60% | 日常互动/温馨/幽默/内心反思 |
| 0 | 常温 | 10-20 | 中 | 30% | 信息传递/过渡/环境建立/说明 |
| -1 | 微冷 | 8-16 | 远 | 10% | 紧张/对峙/悬疑/压抑愤怒 |
| -2 | 极寒 | 6-12 | 极远（旁观） | 0% | 负向高潮：死亡/告别/背叛/重大失去 |

**+2 滚烫技法**：
- 允许内心独白，但必须是口语化
- 感官密度 ≥ 3 种同时出场
- 身体反应写具体——不是"激动"，是"指甲抠进了掌心"
- 句子可以有轻微破碎感，模拟情绪激动时的呼吸节奏
- **禁止**：空洞情绪词堆砌（"他无比激动，万分欣喜"）、感叹号超过1个

**-2 极寒技法**：
- 全部简单动词：站/看/走/摸/放/坐
- 零情绪词、零评论、零内心独白
- 对话标签只用"说"
- **用感官缺席写情感在场**：不说"他心碎了"，写"他把杯子放回桌上。放歪了。"
- 重复一个无意义动作——这是人在巨大冲击下的真实反应
- 场景结尾不加任何总结

**温度切换规则**：

```
允许的切换：
  +2 → 0（高潮后缓冲）    -2 → -1（慢慢回暖）
  +1 → -1（温馨转紧张）    -1 → +1（破冰时刻）
  0 → +1（正常递进）      0 → -1（正常递进）
  
禁止的切换：
  +2 → +2 连续（情感透支，读者麻木）
  -2 → +2 跳跃（冰火两重天，情感失真）
  0 → +2 无铺垫跳跃（没有积累的高潮是自嗨）
  -2 → 0 立即恢复（情感不真实，需要-1过渡）
```

**三档章节的温度曲线模板**：

```
快档章（冲突密集）： 0 → +1 → +2 → -1 → 0
中档章（推进剧情）： +1 → 0 → +1 → 0 → -1 → +1
慢档章（情感深化）： +1 → 0 → -1 → -2 → -1 → 0
```

**操作**：每章写前确定温度曲线，写在章纲顶部。`novel mode generate <master-id>` 自动加载对应大师的温度偏好。

### 3.6 语言指纹锻造——你的个人风格词典

风格系统的终极目标不是永远借用大师，而是**锻造你自己的语言指纹**。

#### 步骤一：参数自诊

从以下维度确定你的自然倾向（不是"想成为谁"，是"你本来怎么写"）：

```
□ 句长偏好：[ ]短句驱动(8-15)  [ ]中句(12-22)  [ ]长句(18-30)
□ 对话占比：[ ]对话为主(>50%)  [ ]均衡(30-50%)  [ ]叙述为主(<30%)
□ 比喻密度：[ ]极少比喻(≤2‰)  [ ]适中(3-6‰)   [ ]密集(≥8‰)
□ 主导感官：[ ]视觉  [ ]听觉  [ ]触觉/体感  [ ]嗅觉/味觉
□ 叙事距离：[ ]极近(读心)  [ ]近  [ ]中  [ ]远  [ ]极远(旁观)
□ 情绪处理：[ ]直接释放(热)  [ ]克制收敛(冷)  [ ]悲喜同句(混合)
□ 语言质地：[ ]口语化  [ ]书面化  [ ]混合
□ 幽默偏好：[ ]冷幽默/逻辑梗  [ ]夸张/滑稽  [ ]基本不搞笑
```

#### 步骤二：提取语言指纹

写 500 字不受任何规则约束的自由文本（一个你最喜欢的场景类型），然后分析：

1. **高频词汇 TOP 10**：除去"的/了/是/我/他/她"
2. **平均句长**：总字数 ÷ 总句数
3. **句长标准差**：句长变化幅度。标准差越大，节奏越丰富
4. **最长句**和**最短句**：这两句各自在什么情境下出现？
5. **对话标签统计**：说了几次"说"？几次"道"？几次其他标签？
6. **感官词汇统计**：视觉/听觉/触觉/嗅觉/味觉各出现多少次？
7. **情绪词密度**：直接写情绪的词（愤怒/悲伤/喜悦/恐惧）÷ 总字数
8. **弱化副词密度**：微微/淡淡/缓缓/轻轻/悄悄 ÷ 总字数

#### 步骤三：定义你的风格边界

```
我的风格 = 自然倾向 + 有意识的选择

核心参数：
  句长范围：____ 到 ____ 字
  对话占比：____%
  比喻密度：____‰（每千字比喻数）
  主导感官：____（选1-2种）
  叙事距离：____
  高潮温度：____（+2 到 -2）
  
标志性技法（你最擅长的 2-3 种）：
  1. _________
  2. _________
  3. _________
  
禁用模式（你的风格中绝不出现在正文的 3-5 样东西）：
  1. _________
  2. _________
  3. _________
  4. _________
  5. _________
```

#### 步骤四：打磨风格金样本

写 500-1000 字能代表你风格最高水平的文本。这段文本：
- 是你的风格基准线——以后每章写完都跟它比对
- 是写前朗读的材料——写前读一遍，锚定语感
- 是风格检测的参考——`novel style check <file> --baseline` 对比偏差

写入 `style_guide.md` 的 `gold_sample` 字段。

### 3.7 五感签名系统

每个作家都有一种主导感官。这是你文字中最独特的信息通道。

**选定你的感官签名**（最多2种）：

| 签名感官 | 适合的作者类型 | 签名技法 | 代表导师 |
|---------|-------------|---------|---------|
| **视觉签名** | 画面感强、善于空间构图 | 精准色彩词、空间透视、光影变化 | 沈从文、刘慈欣 |
| **听觉签名** | 对话高手、节奏感好 | 沉默的重量、声音的层次、节奏控制 | 老舍、王小波 |
| **体感签名** | 身体性、动作戏 | 触觉精确、重量感、温度感、疼痛精度 | 余华 |
| **嗅觉签名** | 氛围营造、记忆唤起 | 气味与记忆绑定、时代气味、人物气味 | 张爱玲 |
| **味觉签名** | 日常感、生活流 | 食物即世界观、味道即情绪、饥饱即阶级 | 汪曾祺 |
| **全能感官** | 史诗感、感官交响 | 五感轮替、在高潮处五感全开 | 莫言 |

**签名感官的使用法则**：
1. 每场景至少出现1次你的签名感官
2. 在情感爆发点，用签名感官取代情绪词
3. 配角可以用签名感官的强弱来暗示重要性（主角注意到某人身上的气味 → 这个人重要）
4. 换场景时切换辅助感官，但签名感官保持稳定

### 3.8 反AI味检测与风格纯度

你的风格最危险的敌人不是写得不好——是**读起来像AI写的**。

#### 12类AI味检测维度

| # | 类别 | 阈值（每3000字） | 检测内容 |
|---|------|---------|---------|

| 2 | 弱化副词 | ≤3/千字 | 微微/淡淡/缓缓/轻轻/悄悄/算是/某种程度上/简直 |
| 3 | 意义膨胀 | 命中即标记 | 意义深远/前所未有/叹为观止/不容忽视/至关重要 |
| 4 | 结论套话 | 命中即标记 | 展望未来/未来可期/总而言之/综上所述/从此以后 |
| 5 | 论文式段落 | ≤25%段 | 段落以总结句开头 / 先总后分的结构 |
| 6 | 正式语体 | 命中即标记 | 于是乎/一方面/另一方面/从而/因而/不仅如此 |

#### 风格模式感知阈值

不同风格导师对AI味的容忍度不同。检测阈值应根据当前激活的风格自动调整：

| 检测规则 | 余华 | 张爱玲 | 王小波 | 莫言 | 老舍 | 汪曾祺 |
|---------|------|--------|--------|------|------|--------|
| 情绪词/千字 | 0 | 3 | 2 | 5 | 2 | 1 |
| 弱化副词/千字 | 0 | 2 | 1 | 3 | 1 | 1 |
| 比喻密度‰ | ≤2 | ≥8 | 3-6 | ≥10 | 2-4 | 1-3 |
| 心理描写 | ❌ | ✅ | ✅ | ✅ | 间接 | ❌ |
| 叙述者评论 | ❌ | 允许 | ✅ | 允许 | ✅ | 少量 |
| 对话标签 | 只说 | 说/问/答 | 说/问 | 说/问/答/喊 | 说/问 | 说/问/答 |

**操作**：
```bash

```

三个命令全部 PASS → 进入下一章。任何一个不通过 → 回到第五卷 Phase 5 修改。

## 第四卷：角色铸造层

> 读者可能忘记情节，但永远不会忘记他们在乎的人。

### 4.1 角色设计卡——每个有名字的角色必填

```yaml
角色名: ___________
功能位: [ ]主角 [ ]反派 [ ]Ally [ ]Mentor [ ]Shadow [ ]Foil [ ]Love [ ]Shapeshifter

# 外在
外貌关键词(≤5个): _________
标志习惯（至少1个，不用每次写但关键场景出现）: _________
标志物品（至少1个，具有叙事功能）: _________

# 内在
Want（想要什么——角色自己知道）: _________
Need（需要什么——角色一开始不知道，成长后明白）: _________
Flaw（阻碍得到Need的东西——最核心的性格缺陷）: _________
Ghost（造就Flaw的源头——通常是创伤记忆）: _________

# 矛盾
内在矛盾（两个互相打架的特质）: _________ vs _________
外在矛盾（与外部世界的冲突）: _________

# 声音（详见4.2）
句长偏好: [ ]短 [ ]中 [ ]长
口头禅/口癖(1-2个): _________
禁用词（绝不说的话）: _________
典型句例(≥3句): _________

# 情绪出口（生气/恐惧/悲伤/开心/撒谎时的具体表现——不能是"他生气了"）
愤怒: _________
恐惧: _________
悲伤: _________
开心: _________
撒谎: _________

# 底牌
最怕的: _________
最想要的: _________
愿意付出的最大代价: _________
不能跨的底线: _________
```

**自检清单**：
- □ Want ≠ Need，两者之间有真正的张力
- □ Flaw 是真的缺陷，不是"太善良了"
- □ 反派有"认为自己才是对的"的理由
- □ Ghost 具体且可信（不是"童年遭遇变故"这种空泛描述）
- □ 情绪出口 ≥ 2 种身体反应（不是"他愤怒地攥紧拳头"这种写法的变体）
- □ 遮名可辨测试通过

### 4.2 七维语音差异化

遮住角色名，纯看对话，能分清谁在说话。这是角色塑造的最低标准，也是最高标准。

| 维度 | 含义 | 设计方法 |
|------|------|---------|
| **句长偏好** | 说话用多长的句子 | A:"走。" B:"我觉得我们可以考虑先离开这里。" |
| **词汇层级** | 雅/俗/文/野/专业 | A:"吃饭" B:"用膳" C:"干饭" D:"补充碳水" |
| **句式习惯** | 陈述/疑问/反问/祈使偏好 | A:全陈述句 B:三句夹一句反问 C:从不说祈使句 |
| **节奏模式** | 连绵还是间断 | A:一口气说完不喘气 B:说一句停一下看对方反应 |
| **口头禅** | 反复出现的词语或句式 | A:"你听我说"（控制型） B:"是吧"（寻求认同） |
| **回避语言** | 绝不说的话 | A:绝不说"爱" B:绝不说"对不起" C:绝不说"怕" |
| **对话策略** | 直来直去还是绕弯子 | A:要什么直接说 B:从不直接说——绕三圈才到主题 |

**有效区分标准**：两个角色至少在 3 个维度上有显著差异，才算有效区分。

**操作**：每写完一场对话，遮名重读。如果可以互换台词而察觉不到 → 回到语音卡重做。

### 4.3 同情绪不同表达——情绪×性格矩阵

同样的情绪，不同性格的角色有完全不同的表达方式：

| 情绪 | 外放型 | 内敛型 | 转移型 | 计算型 |
|------|--------|--------|--------|--------|
| **愤怒** | 脸红/青筋/脏话/砸东西 | 下颌收紧/话变短/语气变冷 | 笑着说的但笑不到眼睛/喝酒/运动发泄 | 表面毫无变化/语气变客气/开始列计划 |
| **恐惧** | 出汗/发抖/音量不自觉变大 | 手心出汗/不说话/嘴唇发干 | 话变多语速变快/反而往前走 | 身体僵硬/大脑空白/动不了 |
| **悲伤** | 哭/哽咽/说不完整句子 | 嗓子堵/话反而变多为了不哭 | 不停做事不让自已停下来 | 什么都没感觉到/一个人待着/发呆 |
| **开心** | 大笑/拍桌子/音量拔高 | 嘴角动了一下/眼睛亮了一下 | 嘴上说"还行吧"但开始哼歌 | 表情不变但效率翻倍 |
| **撒谎** | 眼神飘忽/解释过多/补细节 | 眼神反而更坚定/话变少 | 特别放松/开着玩笑撒谎 | 装傻/绕圈子/不正面回答 |

**硬规则**：
- 角色绝不在对话中说"我生气了""我很难过""我很害怕"
- 配角语音 ≠ 主角语音的降级版（配角是独立的声音，不是主角的劣化版）
- 每角色一张语音卡。≥3维差异才算有效区分
- 每场对话遮名重读

### 4.4 配角功能位设计

配角不是为了"增加角色数"而存在的。每个配角承担至少一个叙事功能：

| 功能位 | 作用 | 设计要点 | 例子 |
|--------|------|---------|------|
| **Ally** | 支持主角但有不同观点 | 必须在某个核心问题上与主角有分歧，不然只是跟班 | 赫敏之于哈利 |
| **Mentor** | 传授能力或智慧 | 必须有自己的盲区，不能在主角成长后仍然无敌 | 邓布利多的盲区 |
| **Shadow** | 主角的另一种可能——最强配角类型 | 和主角有相似的起点/能力，但做了不同的选择 | 言豫津之于梅长苏 |
| **Foil** | 同一情境下做出不同反应 | 用对比来定义主角的性格。没有Foil，主角的特点看不出来 | 鸣人 vs 佐助 |
| **Shapeshifter** | 忠诚不确定的角色 | 读者的怀疑对象。不到最后不揭晓 | 斯内普 |
| **Love** | 情感线 | 必须有独立于主角的目标和个人成长弧。不能只是"主角的奖励" | — |
| **Comic Relief** | 节奏调节 | 在紧张场景之间提供喘息。但不能只是搞笑——要有自己的价值 | — |

**配角设计的三个禁止**：
1. 禁止配角智商不在线——这衬托不出主角的高智商，只会让整个故事看起来蠢
2. 禁止配角的唯一功能是"被主角打脸"——这是最廉价的爽点
3. 禁止配角没有自己的"Want"——哪怕只用一句话定义

### 4.5 知识图谱与关系管理

长篇连载中，角色关系会变得极其复杂。使用知识图谱工具自动化管理：

**关系类型库**（70+ 规范关系类型，6 大类染色）：
- 血缘关系：父子/母女/兄弟/姐妹/叔侄/表亲/……
- 情感关系：爱慕/暗恋/前任/夫妻/暧昧/单恋/……
- 社交关系：师徒/盟友/主仆/同学/同事/邻居/……
- 敌对关系：仇敌/宿敌/情敌/竞争/背叛者/……
- 权力关系：上下级/君臣/主从/……
- 秘密关系：真实身份/隐藏动机/未揭示的关联

**图愈合操作**：
```bash
novel graph heal <project_dir>     # 自动愈合：临时节点毕业/重复合并/孤立清理
novel graph conflicts <project_dir>  # 关系冲突检测（家庭+敌对同时存在→CRITICAL）
novel graph query "related_to:主角 type:暗恋"  # 查询特定关系
```

## 第五卷：执行生产层

> 从灵感到"可以发了"。每一步有输入、有输出、有验证。

### Phase 0：灵感孵化（~30分钟）

**输入**：一个模糊的灵感
**输出**：`idea_seed.md`（含战略定位参数）

1. **一句话种子**：当[主角]遭遇[事件]，必须[行动]，否则[灾难]
2. **母题定位**（见 1.1）：情绪内核 + 赛道映射
3. **欲望三角**：Want（想要什么）/ Obstacle（障碍）/ Cost（代价）/ Need（真正的需要）
4. **逆推三锚点**：终局画面 → 中点转折 → 开场画面（从后往前推导）
5. **平台参数**（见 1.3）：主发平台 + 关键参数设定
6. **五要素定调**：目标读者 / 风格基调 / 创作禁区 / 自动化等级 / 预估规模

### Phase 1：人物塑魂（~60分钟）

**输入**：`idea_seed.md`
**输出**：`character_bible.md` + 全部角色设计卡

每个有名字的角色填写完整设计卡（见 4.1）。主角 + 反派 + 前3配角优先。

**自检**：Want≠Need有张力？反派有"认为自己才是对的"的理由？遮名可辨？配角功能位明确且不重叠？

### Phase 2：大纲规划（~60分钟）

**输入**：角色圣经 + 节拍模板
**输出**：`beat_sheet.md` + `plot_blueprint.md` + `plot_threads.md`

1. **运行**：`novel beat <N> --markdown` 生成节拍骨架
2. **情绪折线**（见 2.1）：为每卷画情绪曲线，标注高低点和情绪类型
3. **四轮驱动**（见 2.2）：确认实力/世界/矛盾/爽点四个引擎的循环
4. **三线钩**（见 2.3）：埋入现实钩（第1章）+ 历史钩（第8章）+ 未来钩（第18章）
5. **难度梯度**（见 2.4）：确认每次挑战递增 20-30%
6. **伏笔计划**：预设 ≥ 5 条跨卷伏笔，写入 `plot_threads.md`

### Phase 3：写前 SOP（~15分钟/章）

**A. 上下文加载**：
- 角色状态刷新：本章开始时每个角色的位置/情绪/目标
- 事实检索（距离衰减）：≤3章前 ×1.0 / 4-10章前 ×0.7 / >10章前 ×0.5
- 伏笔扫描：未回收 > 已回收 ×1.5 → 优先回收
- 世界规则刷新：本章涉及的世界规则确认

**B. 风格锚定**：
- 确定本章温度曲线（见 3.2）
- 确定场景类型 → 匹配导师（见 3.4 场景→导师映射）
- 如果是关键场景：`novel mode generate <master-id>` 获取完整写作指令
- 朗读风格金样本 500 字（见 3.6 步骤四）

**C. 场景澄清五问**：
1. POV角色的 Want 是什么？（本章层面的具体目标）
2. 具体的障碍是什么？（不能是模糊的，要可触摸）
3. 本章结束时情况如何变化？（至少一个有意义的变化）
4. 读者获得什么新信息？（至少 1 条，最好是秘密/线索/角色新面向）
5. 情感弧线是什么？（从 __ 情绪 → __ 情绪。不能是平的）

**D. 高危场景预标注**：
- 情感高潮场景 → 激活反AI味专项检测 + 冷热切换检查
- 打斗/动作场景 → 激活句长 ≤ 15 字检查
- 对话密集型场景 → 激活遮名可辨 + 标签只说/问/答检查
- 揭秘/真相场景 → 激活"不要一次性全揭示"检查

**E-F. 写200-500字概要 → 自审 → 写作授权签发**

### Phase 4：快写（~60-90分钟/章）

**核心心法——导演模式**：
你不是在写。你是在搭一个场景，把角色放进去，然后观察他们做什么，记录下来。

**硬约束**：
- 对话 = 行动（每句对话必须推动/揭示/冲突三者至少其一）
- 遮名可辨（每写完一场对话就自测）
- 情绪不空洞（身体反应 + 物质细节，不用情绪词）
- 句长可变（不连续 5 句同长度）
- 对话标签只说/问/答。偶尔动作打断

**快写期间不要**：
- 不要回头修改（所有修改留到 Phase 5-7）
- 不要查资料（用 [TODO: 查XX] 标记，事后补）
- 不要纠结措辞（先让故事流出来，措辞是 Phase 7 的事）

### Phase 5：热叙事三箭（~30分钟/章）

**第一箭·情绪强化**：
- 每个情感节点 → 身体反应 ≥ 2 种？
- 大悲/大痛 → 冷叙事（句长 6-12 字 / 简单动词 / 零度标签 / 感官缺席写在场）
- 大爽/大喜 → 感官洪流（≥ 3 种感官同时 / 允许口语化内心独白）

**第二箭·口语检测**：
- 能念出来吗？（默念每一段）
- 每句话有"说话的人"吗？（不会出现作者的声音混进角色视角）
- 夹带作者文采了吗？（如果有——删掉，把文采留给诗歌，小说要的是真实）
- 标签全是"说/问/答"吗？

**第三箭·多角色差异化**：
- 遮名可辨？
- 同事件各角色反应不同？（Foil 的有效性检测）
- 各有自己的情绪出口？（不是所有人都用同一种方式表达同一情绪）
- 配角有独立于主角的目标吗？

### Phase 6：增色（~45分钟/章）

- **感官补全**：每场景 ≥ 1 种非视觉感官 + 你的签名感官 ≥ 1 次（见 3.7）
- **物质细节**：每 300 字 ≥ 1 个具体物质（必须具有叙事功能，不是纯粹的装饰性描写）
- **段落节奏**：长段（≥200字）后用短段（≤30字）切断。连续长段不超过3个
- **对话节奏**：对话中插入非对话打断（动作/环境/沉默）
- **比喻审核**：每个比喻都问——它帮助读者理解了什么？没有叙事功能的比喻 → 删

**自检**：
- 对话有冲突（含潜冲突）吗？如果没有，这场对话在干什么？
- 动作句长 ≤ 15 吗？如果超过，拆开
- 心理描写太多情绪词了吗？转成身体反应
- 环境描写有叙事功能吗？纯粹的氛围描写至少要有情绪投射功能

### Phase 7：精修（~30分钟/章）

1. **开头第一句够不够好？**不铺垫，不描写，第一句就让人想读第二句
2. **结尾制造翻页欲望了吗？**（悬念/新信息/情绪余韵/一句扎心的话）
3. **有可以删的段落吗？**→ 找到就删。删了不妨碍理解的部分都是多余的
4. **读一遍——哪段让你觉得无聊？**→ 那段要么删，要么往里面注入冲突/信息/情绪

### Phase 8：质量门禁（~10分钟/章）

```bash
# AI味检测
novel deslop scan ch_N.md          # 必须: CRITICAL=0 ∧ HIGH≤2

# 文笔规则
novel style lint ch_N.md            # 必须: HIGH≤3

# 风格一致性（如指定了风格模式）
novel mode apply <master> ch_N.md   # 必须: CRITICAL=0
```

→ **全部 PASS → 批准进入下一章**
→ 任何一项不通过 → 回到 Phase 5，针对性修改后重新走门禁

### Phase 9：里程碑体检（每 10 章）

```bash
# 质量总评
novel score milestone <dir>         # Q-Score ≥ 8.0（见下文）

# 一致性
novel entropy batch <dir>           # ETC ≤ 2.0 ∧ EWC ≤ 1.5

# 知识审计

# 全书文笔一致性

```

**Q-Score 八维评分**：
①情节结构 20% | ②角色塑造 18% | ③文笔语言 12% | ④世界观 10% | ⑤主题思想 8% | ⑥情感冲击 12% | ⑦阅读体验 12% | ⑧预期管理 8%

**里程碑健康指标**：
- 连续下滑（任一维度 3 次连续体检 ↓）→ 暂停，分析原因
- 情感疲乏（连续 5 章同一情绪类型）→ 调节奏
- 伏笔堆积（未回收 > 已回收 ×1.5）→ 暂停新伏笔，优先回收
- 角色闲置（重要配角 10 章未出场）→ 安排出场或考虑合并角色

## 第六卷：商业运营层

> 写完不是结束。让作品找到读者，让读者成为粉丝。

### 6.1 黄金开篇——致命 300 字

开篇前 300 字决定读者是翻页还是划走。

**三重爆破结构**：
```
第1爆·情境爆破（第1-50字）
  → 一个具体的、不寻常的、正在发生的画面
  → 不是介绍，是直接扔进场景
  → 例："天台上，校霸的烟头掉在了我的运动鞋上。"
  → 不是："我叫林默，是江城一中的高二学生。"

第2爆·情绪爆破（第50-150字）
  → 主角的即时生理反应
  → 不是"她感到害怕"，是"她的膝盖在抖"
  → 启动读者的镜像神经元

第3爆·悬念爆破（第150-300字）
  → 一个微小的异常——暗示有更大的事在发生
  → 不是"这个世界隐藏着秘密"，是一个具体的、可触摸的异常
  → 例："烟头明明落在鞋上。但鞋没烫出洞。"
```

**前三章必须包含的元素**（顺序可调，但缺一不可）：
1. 背景（主角日常 + 缺憾）
2. 危机（打破日常的事件）
3. 金手指（不寻常的能力/信息/资源）
4. 金手指解决危机（展示金手指的价值）
5. 收获（第一个小爽点）
6. 新的钩子（更大的悬念/目标）

**开篇禁忌**：
- 开篇出现一堆人名（读者记不住也不想记）
- 前世/背景介绍超过 100 字
- 主角独白式自我介绍
- 世界观说明文（"这个世界分为九大洲……" → 删，没人读）
- 无关人物堆砌（开篇只聚焦主角 1 人，配角第3章后再引入）

### 6.2 章纲功能卡

不写流水账章纲。按功能模块切分每章：

```
章纲设计公式（来自飞卢编辑实战经验）：

矛盾起因 → 小拉扯 → 初步告捷 → 继续拉扯 → 拉扯成功 → 拉扯影响 → 新矛盾诞生

每个阶段过渡时必做：
  1. 盘点胜利果实（主角得到了什么？）
  2. 梳理角色关系（谁的态度变了？）
  3. 更正关系变化（谁从敌人变盟友？谁反之？）
```

**功能卡实例**：

```
【第15章 功能卡】
标题：_____
节奏档：[ ]快 [ ]中 [ ]慢
温度曲线：0 → +1 → -1 → +2 → 0
POV：主角 80% + Ally视角 20%（过渡段）

功能模块：
  [Hook] 接上章悬念 —— 实验室的门开了，里面是 _____
  [Conflict-1] 主角 vs 守卫 —— 武力对抗（展示新获得的技能）
  [Breather] Ally 的吐槽 —— 调节节奏，展示两人关系
  [Conflict-2] 主角 vs 自己的恐惧 —— 内心的声音劝他放弃
  [Mini-win] 突破第一道防线 —— 获得关键线索
  [Setback] 线索指向更大的阴谋 —— 原来这只是冰山一角
  [Cliffhanger] 神秘人出现 —— "你终于来了。我等了你三年。"

爽点类型：[ ]升级 [√]解密 [ ]打脸 [ ]收集 [ ]养成 [ ]感情
情绪价值密度：HIGH（冲突 + 解密 + 情感三位一体）
```

### 6.3 五分钟情绪周期

来自番茄畅销榜的数据规律：读者平均每 5 分钟需要一次情绪刺激。

```
3000字章节的情绪周期设计：
  0-300字：开场钩子（接上章悬念 或 新冲突）
  300-600字：第一次微型冲突或信息增量
  600-900字：第一次情绪回报（小爽/小暖/小惊讶）
  900-1500字：主要冲突展开
  1500-1800字：第二次微型冲突
  1800-2100字：中期转折（情况比想的复杂/简单）
  2100-2500字：第三次微型冲突
  2500-2700字：情绪高潮（本场最大爽点或最深情感）
  2700-3000字：收尾 + 新钩子
```

**情绪类型轮换表**（防止连续使用同一类型）：
```
Chapter N:   悬念→惊讶→满足→紧张→悬念
Chapter N+1: 温馨→紧张→轻松→感动→悬念（换情绪类型）
Chapter N+2: 幽默→紧张→惊讶→震撼→悬念（再换）
```

### 6.4 爽点工程学

爽点的本质不是"主角赢了"，而是**读者的大脑在说"对！就该这样！"**。

**因果链条三律**：

| 定律 | 含义 | 违反示例 |
|------|------|---------|
| **动机律** | 爽点行为必须符合主角的核心欲望 | 一个只想种田的主角突然去争霸 → 前后矛盾 |
| **能力律** | 爽点行为在主角能力范围内（经过铺垫的能力） | 不越级秒杀。越级秒杀=爽点贬值 |
| **后果律** | 爽点行为触发有意义的连锁反应 | 打了反派的脸，下一章什么事都没发生 → 爽感清零 |

**爽点放大器——吃瓜群众的反应**：
- 反派越跳，打脸越爽
- 大佬越震惊，主角越牛逼
- 围观群众的反应层级：怀疑→震惊→敬畏→狂热
- 提前埋伏的反差（"昨天还看不起他的人，今天跪着求他"）

**爽点释放四步曲**：
```
压抑（让读者感到"憋屈"）
  ↓
积累（让憋屈持续一会儿——但不能太久）
  ↓
爆发（集中释放——节奏加快、句长缩短、感官密度增加）
  ↓
收尾（1.5-2章日常舒缓节奏，让读者消化。没有收尾 → 高潮贬值）
```

**高潮节奏控制**：
- 高潮前：节奏加速（句子变短 / 段落变短 / 对话变密集）
- 高潮中：全章集中，不切换 POV，不插回忆
- 高潮后：1.5-2 章的日常缓冲，让读者喘息
- 慎用连续高潮——两场高潮之间必须有足够的低压区

### 6.5 数据化校准——读者心理博弈

写完不是结束。数据反馈才是真正的开始。

**核心指标**：
- **追读率** > 收藏量（追读 = 明天还会看的真实读者数）
- **完读率** > 点击量（完读 = 读者真的读完了，不是点进来就划走）
- **章说互动率** = 章说数 ÷ 追读数（互动率 > 5% → 读者有情绪投入）

**数据下滑诊断手册**：

| 症状 | 诊断 | 处方 |
|------|------|------|
| 30章后完读率腰斩 | 过度描写无关支线 / 主角目标模糊 | 回去看 2.2 节四轮驱动——是不是某个引擎停了 |
| 上架后追读暴跌 | 前面免费章的爽点透支了后面的期待 | 检查免费章的最后3章——是不是把最核心的钩子提前用掉了 |
| 章说开始骂娘 | 触碰平台毒点（男频绿帽/女频种马/主角降智） | 立即检查最近5章——找侮辱读者智商的情节 |
| 追读稳定但不涨 | 缺乏破圈钩子，只有老读者在追 | 检查标题和简介——是不是只有老读者看得懂 |
| 完读率高但章说少 | 读者在看但不投入感情 | 增加角色"被误解"和"选择时刻"——让读者有站队欲望 |

**差评处理黄金法则**：
- 黄金 2 小时内回复可转化 45% 的差评读者
- 章说回复控制在 3-5 条/天 → 保持神秘感
- 绝对不要跟读者争辩剧情走向 → "谢谢你的建议，后面会有交代"
- 差评中出现 3 次以上同样的批评 → 这可能不是差评，是诊断信号

### 6.6 防崩盘机制——长线运营

**存稿纪律**：
- 至少储备 10 章存稿再开始发文
- 日更 4000-6000 字是算法敏感区间（低于 4000 降权，高于 6000 边际收益递减）
- 专注拼字法（唐家三少方法论）：遇到难写的段落跳过，先写后面的，事后补

**换地图机制**：
换地图不是因为写腻了，而是因为**这张地图的矛盾已经写光了**。
- 判断标准：主角在这张地图上已经没有同级别的对手 / 所有伏笔都已揭示 / 所有关系线都已展开
- 换地图时同步升级：对手层级 + 1 / 资源稀缺度 + 1 / 秘密深度 + 1 / 势力复杂度 + 1
- 塔状地图模型：城市 → 区域 → 大陆 → 星域 → 宇宙
- 每层地图的章节分配：约当前总章节的 30-40%

**矛盾承载力**：
每张地图的矛盾承载力有限。判断标准：
- 主角已经无敌于此地图 → 矛盾清零 → 必须换地图
- 读者开始问"为什么主角不去更大的地方" → 矛盾即将清零 → 准备换地图
- 连续 5 章的爽点都在"秒杀弱对手" → 已经在消耗存量矛盾 → 尽快换地图

### 6.7 2026 未来趋势接口

**多元宇宙叙事标配**：
现世线（日常真实感）+ 灵界线（奇幻猎奇）+ 记忆线（悬疑解谜），三界穿梭。每界的规则不同，主角需要在不同规则间切换生存策略。

**从"升级循环"到"认知循环"**：
主角不只是变强，而是对世界的理解每次循环都更深一层。每次回到起点，收获的不是新技能，是新真相和新代价。

**技术融合文本实验**：
- AR 线索解密：在现实场景中嵌入小说暗线（二维码→隐藏章节 / 实景照片→剧情线索）
- 弹幕选择：在关键节点让读者投票决定剧情走向
- 跨媒介叙事：同一个故事在小说 + 短视频 + 互动游戏中以不同视角展开

**AI 时代作者的核心竞争力三要素**：
1. **个人风格不可替代性**（第三卷的全部内容 → 这是 AI 最难复制的）
2. **情感真实性**（你的人生母题 → AI 没有活过，不知道什么是疼）
3. **读者社区运营力**（你的粉丝不是读者，是"跟你一起创造世界的人"）

## 附录 E：短篇/中篇创作轨道 ★ v2.0 新增

> 长篇系统完整保留。本附录是针对知乎盐选、豆瓣阅读等短中篇投稿平台的精简适配。

### E.1 长篇 vs 短篇：系统的取舍

创世银河是为 50 万字+长篇连载设计的。短篇（5000-30000字）的逻辑有根本不同：

| 长篇需要（保留） | 短篇不需要（丢弃） | 原因 |
|----------------|------------------|------|
| 第二卷·四轮驱动引擎 | ✂️ 全部丢弃 | 3万字写完就结束了，不会"崩盘" |
| 2.1 情绪抑扬波（卷级） | ✂️ 丢弃 | 短篇只有一个核心波峰（反转点） |
| 2.4 难度梯度 20-30% | ✂️ 丢弃 | 短篇通常只有 1-2 个核心冲突 |
| 2.5 15节拍法 | ✂️ 替换 | 短篇用 3-5 节拍结构 |
| 第六卷 6.2-6.6（连载运营） | ✂️ 丢弃 | 没有"连载中"的概念 |
| 第五卷 Phase 9（里程碑体检） | ✂️ 丢弃 | 短篇不需要 10 章体检 |
| **第三卷·风格锻造（全部）** | ✅ 保留并强化 | 短篇字数少→每句话的纯度要求比长篇更高 |
| 铁律×5 + 反AI味检测 | ✅ 保留并强化 | 短篇读者更挑剔文字——一票否决 |
| 第一卷·母题锁定 | ✅ 保留 | 短篇需要更快找到情感内核 |
| 第六卷·黄金开篇 | ✅ 保留并强化 | 短篇开篇 = 生死线，比长篇更致命 |
| 第四卷·角色设计（简化版） | ✅ 保留精简版 | 短篇通常 1-3 个核心角色 |

### E.2 知乎盐选 vs 豆瓣阅读：两个完全不同的战场

#### 知乎盐选

| 参数 | 设定 |
|------|------|
| **字数** | 8000-30000字（最佳区间 12000-18000字） |
| **核心驱动** | 概念驱动。一个好的知乎短篇 = 一个可一句话概括的概念 + 一个读者猜不到但回想起来合理的反转 + "这像真的发生过"的叙述质感 |
| **读者情绪** | "卧槽还能这样？！" —— 智力被震撼的快感 |
| **叙事视角** | 第一人称优先。"我"让代入更快 |
| **开头公式** | 不是"300字三爆"——知乎是**"前三段=一个完整的微缩反转"**。读者点"展开全文"之前的前三行决定生死 |
| **语言门槛** | 比番茄高，比晋江低。需要"看起来像真的"——知乎读者对真实感有执念 |
| **热门赛道** | 规则怪谈 / 时间循环 / 社会推理 / 职业内幕 / 历史IF线 / 情感反转 / 科幻概念 |
| **变现路径** | 先在自己账号发表→数据好→编辑主动联系。或直接投盐选编辑邮箱。单篇爆款可持续产生收入 |
| **核心指标** | 赞同数 / 收藏数 / 展开率（前三段→全文的转化率，这是最关键的） |

#### 知乎短篇专用结构：3-5 节拍模型

长篇用15节拍，短篇用这个：

```
节拍1·日常建立（10%篇幅）
  → 建立主角的"正常状态"——让读者知道这个人是谁
  → 暗示一个微小的"不对劲"——埋在背景里，不要明说

节拍2·异常发生（20%篇幅）
  → 打破日常的具体事件
  → 主角的即时生理反应（不是"震惊"，是具体身体反应）

节拍3·悬念升级（30%篇幅）
  → 异常逐步扩大/加深，读者开始猜
  → 主角的主动调查/应对——不能是被动等待
  → 埋下反转的伏笔（不能太明显，但回头翻要能找到）

节拍4·反转揭示（25%篇幅）
  → 冲击性真相。可以是单一反转或双重反转
  → 反转必须满足：出人意料 + 回想合理 + 情感冲击
  → 禁止：反转来自读者不知道的新信息（作弊）

节拍5·余韵收尾（15%篇幅）
  → 不解释反转（让读者自己想）
  → 回到日常——但日常已经变了颜色
  → 最后一句话留有余味
```

#### 知乎前三段法则（最关键的一步）

知乎的展示逻辑是：正文前三段直接显示在时间线上，后面需要点"展开全文"。**前三段决定生死。**

```
不要这样写（正文式展开）：
  "我叫林夏，高三学生。今天是我人生中最重要的考试。我坐在考场里，
   等着监考员发卷子。窗外蝉鸣很响。我的好闺蜜宋婉坐在前面两排……"

要这样写（倒叙挤压式）：
  "我死在高考考场里。然后我醒了——距离交卷还有30秒。
   我的'好闺蜜'正对着我露出担忧又无辜的眼神。我的书包里，
   躺着她昨晚塞进来的栽赃手机。
   我只有30秒。而我已经死过一次了。"

区别：
  正文式 = 日常→异常→悬念（线性，慢）
  知乎式 = 异常→结果→问题（倒叙，把最炸的信息放在最前面）
```

**前三段写作规则**：
- 第一段 = 一个不正常的、正在进行中的具体画面或陈述。不铺垫。
- 第二段 = 为什么这个画面不对劲——提供冲突的线索
- 第三段 = 一个让读者必须点"展开全文"的钩子（问题/悬念/情感悬念）

#### 豆瓣阅读

| 参数 | 设定 |
|------|------|
| **字数** | 5000-20000字 |
| **核心驱动** | 情感/氛围驱动。一个好的豆瓣短篇 = 一个读者看完会说"对，就是这样"的瞬间 + 一段读起来不费劲但有回味的文字 + 一个人物让读者想起自己或身边的人 |
| **读者情绪** | "对……就是这样。" —— 被理解、被看见的共鸣 |
| **叙事视角** | 第一/第三人称均可。第三人称更安全 |
| **语言门槛** | 比知乎高。需要文学性但不能"端着"。最怕——装文艺 |
| **热门赛道** | 都市情感 / 成长 / 家庭关系 / 女性 / 边缘群体 / 乡土 / 年代 / 悬疑（偏文学向） |
| **变现路径** | 豆瓣阅读征文大赛 / 自出版 / 签约作者。门槛比知乎高，单篇收益天花板也高 |
| **核心指标** | 编辑推荐 / 读者评分 / 征文入围 |

#### 一句话区分两个平台

| | 知乎盐选 | 豆瓣阅读 |
|------|---------|---------|
| 读者情绪 | "卧槽还能这样？！" | "对……就是这样。" |
| 驱动引擎 | 概念 → 反转（逻辑的震撼） | 人物 → 情感真相（共鸣的震撼） |
| 评判标准 | 聪明：逻辑自洽+意想不到 | 准确：情感真实+不煽情 |
| 失败模式 | 反转太硬/逻辑不自洽 | 无病呻吟/装文艺 |
| 最佳篇幅 | 12000-18000字 | 5000-15000字 |
| 典型开篇 | 前三段微缩反转 | 一个精准的物质细节或身体感受 |

### E.3 短篇创作精简流程（6 Phase）

```
Phase 0: 概念孵化（30分钟）
  输出: 一句话概念 + 反转设计 + 知乎前三段/豆瓣开头段
  
  知乎概念检查：
    □ 能一句话概括吗？（"当[主角]发现[异常]，必须[行动]，否则[灾难]"）
    □ 反转能让人"再也回不去"吗？（知道真相后重读，前面所有细节都有新解释）
    □ 逻辑闭环了吗？（没有靠"读者不知道的信息"来反转——那是作弊）
  
  豆瓣概念检查：
    □ 这个情感是"具体"的吗？（不是"爱情"——是"分手三年后在地铁上闻到他的洗衣液味道"）
    □ 有至少一个"读者会想起自己"的细节吗？
    □ 结尾不是"总结"，是"余韵"吗？

Phase 1: 人物速写（20分钟）
  只做 1-2 个角色的简化设计卡：
    - 主角：Want / Need / Flaw / Ghost / 情绪出口（愤怒/恐惧/悲伤各1种身体反应）
    - 关键配角（如有）：功能位 + 与主角的核心差异（≥3维语音差异）
  
  短篇角色铁律：
    - 不超过 3 个有名有姓的角色（其余用"监考员""门卫"代称）
    - 每个角色的对话遮名可辨（系统4.2节标准不降低）
    - 反派的动机不必展开但必须"可理解"（读者能猜到就够了）

Phase 2: 故事骨架（30分钟）
  输出: 3-5 节拍表（见 E.2）+ 关键场景列表
  
  知乎附加步骤：写三版前三段，选最好的一个
    - 版本A：从最高悬念点切入（最炸但风险最大）
    - 版本B：从一个具体的物质细节切入（最稳）
    - 版本C：从一句"看起来像真的"的陈述切入（最知乎）

Phase 3: 快写（2-4小时）
  一口气写完初稿。不能停，不能改，不能查资料。[TODO]标记事后补。
  
  短篇快写特别规则：
    - 先写结尾段（知道你最后要落在哪，前面才写得对）
    - 5000字以下一篇写完，10000字以上可以分2-3次
    - 写的过程中如果发现反转不够炸——继续写，写完再改。不要在快写阶段推翻重来

Phase 4: 风格精修（1-2小时）
  逐条检查（按系统第三卷标准）：
    1. 铁律一：AI高频词清零。特别注意"眼神/唇角/眸色/不由得/似乎/仿佛"（短篇密度高，比长篇更刺眼）
    2. 铁律二：对话标签都是"说/问/答"吗？
    3. 铁律三：每个情绪点→身体反应≥2种？"大脑一片空白""眼泪决堤"→替换
    4. 铁律四：默念全篇。不顺口→改
    5. 铁律五：遮名可辨测试
    6. 温度检查：高潮段温度选择合理？最痛处用最冷笔法？
    7. 感官配比：签名感官≥1次/500字？非视觉感官≥1次/场景？
    8. 句长可变：没有连续5句同长度？

Phase 5: 开篇重写（20分钟）
  初稿写完后，回过头重新写开篇前三段。
  
  为什么快写完才重写开篇？
  → 初稿的开头通常是"热身"，写完全文才知道真正的故事入口在哪里
  → 最好的开篇往往是初稿中间某一段的后半截，被你从原位移到最前面

Phase 6: 终审门禁（15分钟）
  模拟质量门禁：
    □ 全文朗读一遍——有让你想跳过的段落吗？→ 删或改
    □ 反转是"回想起来合理"的，还是"靠新信息强行反转"的？
    □ 最后一句话——读完之后会在读者脑子里回响吗？
    □ 如果是知乎：前三段做到了"微缩反转"吗？
    □ 如果是豆瓣：有一个"读者会想起自己"的精准细节吗？
```

### E.4 平台选择决策树

```
你的母题是"被背叛后的反击"或"规则内的反杀"？
  → 知乎（概念驱动、反转爽感、逻辑博弈）

你的母题是"无力感与守护"或"被误解的孤独"？
  → 豆瓣（情感驱动、人物共鸣、文学质感）

你的开篇是一个"炸裂的概念"？
  → 知乎（"我死在考场上。然后醒了。"）

你的开篇是一个"精准的感受"？
  → 豆瓣（"地铁到站时，风带来的洗衣液味道。是他以前用的那种。"）

不确定？
  → 先写知乎。反馈更快（当天数据），试错成本更低。
    跑通 3-5 篇找到自己的声音后，再决定是否进豆瓣。
```

### E.5 短篇 vs 长篇：时间投入对比

| | 长篇（50万字） | 短篇（1.5万字） |
|------|------------|------------|
| 创作周期 | 6-12个月 | 3-7天 |
| 反馈周期 | 连载3个月后才能判断成败 | 发布当天看到数据 |
| 试错成本 | 极高（半年写废=沉默成本巨大） | 极低（一周写废=学到了东西） |
| 风格实验 | 切换成本高 | 每篇可以试不同风格 |
| 变现模式 | 连载订阅/IP改编（大后期） | 盐选分成/征文奖金（快速回馈） |
| 系统使用 | 全六卷 | 精简6 Phase |

**建议策略**：用短篇练风格和积累读者，用长篇吃 IP 红利。3-5篇短篇→找到自己最擅长的声音→如果某篇数据爆了→考虑扩展为长篇。
